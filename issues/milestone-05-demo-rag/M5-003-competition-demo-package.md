## 설명

경진대회 제출물은 코드가 동작하는 것만으로는 부족하다. 심사 당일 네트워크, 기기, 센서, 발표 시간이
흔들려도 핵심 메시지를 보여줄 수 있어야 한다. 이 이슈는 실행 체크리스트, 시연 대본, fallback 자료,
문서 정리를 하나로 묶는 마무리 작업이다.

## 작업 내용

### 1. 데모 실행 체크리스트

- 서버 실행 명령, 앱 실행 명령, 필요한 기기/권한을 정리한다.
- 시연 전 확인할 항목을 순서대로 적는다.
- 네트워크 없이 가능한 기능과 서버가 필요한 기능을 구분한다.

### 2. 시연 대본

- 실외 GPS 화면에서 시작한다.
- 입구 진입 후 실내 전환을 보여준다.
- 실내 위치 추적과 경로 안내를 보여준다.
- 자연어 목적지 검색이 준비됐으면 마지막에 짧게 보여준다.
- 실패 시 사전 녹화/스크린샷으로 대체할 흐름을 적는다.

### 3. 백업 자료

- 정상 동작 영상 또는 화면 캡처를 준비한다.
- 평가 그래프와 핵심 수치를 발표용으로 정리한다.
- "인프라 0", "평면도 기반 보정", "자연어 목적지" 메시지를 짧게 정리한다.

### 4. 문서 정리

- 루트 README가 실제 실행법과 프로젝트 차별점을 설명하도록 갱신한다.
- `docs/navigation-overview.md`의 결정 사항을 최신 상태로 맞춘다.
- 필요하면 `VERSION.md`에 데모 버전 태그를 기록한다.
- CD workflow와 Playwright E2E 실행 위치는 M5-004로 연결한다.

## 수용 기준

- README만 보고 시연 환경을 재현할 수 있다.
- 발표자가 따라 읽을 수 있는 3~5분 시연 대본이 있다.
- 라이브 시연 실패 시 보여줄 영상/스크린샷/그래프가 준비되어 있다.
- 프로젝트의 범위, 한계, 차별점이 문서에 정리되어 있다.
- M5-004의 CD/E2E 체크리스트와 충돌하지 않게 배포 대상과 데모 URL이 정리되어 있다.
