## 설명

길찾기는 PDR과 별개로 먼저 정적 데이터 위에서 동작해야 한다. 이 이슈는 M2-001의 route node/edge를
사용해 시작점에서 목적지 POI까지 기본 최단 경로를 계산하고, M2-002의 평면도 위에 경로 선을 표시한다.

초기 구현은 A*나 Dijkstra 중 단순한 쪽으로 시작한다. 중요한 것은 알고리즘 완성도보다 "경로 데이터가
지도 화면에 실제로 그려진다"는 수직 흐름이다.

## 작업 내용

### 1. 경로 그래프 로딩

- 층 GeoJSON의 `node`, `edge` Feature를 클라이언트 모델로 변환한다.
- POI와 가장 가까운 node를 연결하는 규칙을 정한다.
- 간선 가중치는 기본적으로 거리 기반으로 둔다.

### 2. 최단 경로 계산

- `client/lib/navigation/route_planner.dart`에 기본 경로 계산 로직을 만든다.
- 시작점은 입구 또는 더미 현재 위치에서 가장 가까운 node로 잡는다.
- 목적지는 사용자가 선택한 POI에 가장 가까운 node로 잡는다.
- 경로가 없을 때는 빈 결과와 사용자 메시지를 반환한다.

### 3. 화면 표시

- POI를 탭하면 목적지로 선택한다.
- 계산된 경로를 평면도 위 polyline으로 표시한다.
- 거리와 예상 소요 시간을 하단 영역에 표시한다.

### 4. 테스트

- 작은 그래프 fixture로 최단 경로 계산 단위 테스트를 작성한다.
- 경로 없음, 자기 자신 목적지, 간선 가중치가 다른 경우를 확인한다.

## 수용 기준

- 앱에서 POI를 선택하면 평면도 위에 경로 선이 표시된다.
- 경로 거리와 예상 시간이 화면에 표시된다.
- route planner 단위 테스트가 통과한다.
- 경로가 없는 경우 앱이 크래시하지 않고 안내 문구를 보여준다.
