## 설명

데모의 첫 인상은 평면도 화면에서 결정된다. 이 이슈는 FastAPI에서 받은 층 GeoJSON을 Flutter에서
읽어 벽, 복도, POI, 현재 선택 층을 화면에 그리는 작업이다.

아직 실제 현재 위치는 센서로 움직이지 않아도 된다. M2에서는 정적 평면도와 더미 현재 위치만으로
이후 경로 안내 화면의 기반을 만든다.

## 작업 내용

### 1. 데이터 모델 연결

- `client/lib/data/models/`에 `Building`, `FloorPlan`, `Poi`, `RouteNode`, `RouteEdge` 모델을 둔다.
- M2-001의 API 응답을 Dart 모델로 파싱한다.
- 파싱 실패, 빈 데이터, 서버 연결 실패 상태를 구분한다.

### 2. 평면도 레이어 구현

- `features/map/`에서 층 GeoJSON을 받아 벽과 복도를 그린다.
- POI는 아이콘 또는 작은 라벨로 표시한다.
- 더미 현재 위치 마커를 입구 주변에 표시한다.
- 지도 영역은 앱 화면의 중심이 되도록 유지한다.

### 3. 층/건물 상태 UI

- 상단에 건물명과 현재 층을 표시한다.
- 층이 2개 이상이면 층 선택 버튼 또는 간단한 segmented control을 제공한다.
- 로딩/에러/빈 상태를 지도 위에 과하게 가리지 않는 방식으로 표시한다.

### 4. 문서화

- `client/README.md`에 서버 실행 후 지도 데이터를 불러오는 확인 절차를 적는다.
- 샘플 데이터가 바뀌면 화면에서 무엇을 확인해야 하는지 간단히 남긴다.

## 수용 기준

- 서버를 띄운 상태에서 앱을 실행하면 데모 건물 평면도가 표시된다.
- 벽/복도/POI가 서로 다른 시각 요소로 구분된다.
- 층 전환 시 해당 층 데이터로 화면이 바뀐다.
- 서버 연결 실패 시 앱이 크래시하지 않고 에러 상태를 보여준다.
- `flutter analyze`가 통과한다.
