## 설명

Navigation의 시연 임팩트는 사용자가 건물 입구로 들어갈 때 앱이 자동으로 실내 모드로 바뀌는 순간이다.
이 이슈는 입구 좌표 반경 진입과 GPS accuracy 악화를 조합해 실내 진입을 감지하고, 입구 좌표와
heading을 PDR/PF 초기값으로 전달한다.

1차 목표는 안정적인 데모다. 복잡한 ML 기반 indoor/outdoor detection은 후순위로 둔다.

## 작업 내용

### 1. 입구 데이터 사용

- M2-001의 `entrances` 데이터를 클라이언트 모델로 읽는다.
- 현재 GPS 위치와 입구 좌표의 거리를 계산한다.
- 입구 반경 값을 상수로 분리한다.

### 2. 전환 감지

- 사용자가 입구 반경 안으로 들어오면 전환 준비 상태로 바꾼다.
- GPS accuracy 값이 일정 기준 이상 악화되면 실내 진입 후보로 판단한다.
- 짧은 시간 안에 조건이 유지될 때만 실내 진입을 확정한다.

### 3. 초기화 연결

- 실내 진입 확정 시 입구 좌표를 실내 로컬 좌표의 시작점으로 변환한다.
- 진입 직전 GPS 이동 방향 또는 entrance의 `heading_hint`를 초기 heading으로 사용한다.
- PDR 엔진과 Particle Filter 초기화 이벤트를 발생시킨다.

### 4. UI 피드백

- "건물 감지 중", "실내 모드로 전환합니다" 같은 짧은 상태를 표시한다.
- 수동 전환 fallback 버튼을 개발 모드 또는 설정에 둔다.

## 수용 기준

- 입구 근처에서 GPS 조건이 만족되면 실내 모드로 전환된다.
- 전환 시 PDR/PF 초기 위치와 heading이 설정된다.
- 전환 상태가 화면에서 명확하게 보인다.
- 조건이 만족되지 않으면 실외 지도 상태를 유지한다.
