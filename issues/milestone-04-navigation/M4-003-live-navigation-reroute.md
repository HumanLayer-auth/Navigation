## 설명

M2의 정적 경로와 M4의 보정된 현재 위치를 합쳐 실제 내비게이션 화면을 만든다.
사용자가 움직이면 현재 위치 마커가 갱신되고, 목적지까지 남은 경로와 안내 문구가 함께 바뀌어야 한다.

이 이슈까지 끝나면 RAG 없이도 발표 가능한 핵심 데모가 완성된다.

## 작업 내용

### 1. 내비게이션 상태 모델

- `idle`, `selectingDestination`, `navigating`, `arrived`, `offRoute` 상태를 정의한다.
- 현재 위치, 목적지, 경로 polyline, 남은 거리, 다음 안내 문구를 한 상태에서 관리한다.

### 2. 경로 진행 표시

- 현재 위치와 목적지 사이의 경로를 표시한다.
- 지나간 구간과 남은 구간을 구분한다.
- 남은 거리와 예상 시간을 업데이트한다.

### 3. 이탈 감지와 재탐색

- 현재 위치가 경로에서 일정 거리 이상 벗어나면 `offRoute`로 판단한다.
- 가장 가까운 route node를 시작점으로 경로를 다시 계산한다.
- 화면에 "경로를 재탐색합니다" 피드백을 보여준다.

### 4. 도착 처리

- 목적지 반경 안에 들어오면 `arrived` 상태로 바꾼다.
- 도착 오버레이 또는 완료 메시지를 표시한다.
- 새 목적지를 다시 선택할 수 있게 한다.

## 수용 기준

- 목적지를 선택하고 이동하면 현재 위치와 경로 안내가 함께 갱신된다.
- 경로 이탈 시 재탐색이 동작한다.
- 도착 반경 진입 시 도착 상태가 표시된다.
- 센서가 불안정해도 앱이 크래시하지 않는다.
