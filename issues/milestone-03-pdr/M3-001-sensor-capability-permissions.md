## 설명

PDR은 센서가 없으면 성립하지 않는다. 이 이슈는 앱 실행 시 가속도계, 자이로, 지자기, 기압계,
위치 권한 상태를 확인하고, 지원되지 않는 기기에서도 앱이 안전하게 제한 모드로 동작하도록 만든다.

경진대회 데모는 풀 지원 기기로 진행하더라도, 제한 조건을 명확히 보여주는 것은 프로젝트의 신뢰도를
높인다.

## 작업 내용

### 1. 센서 capability 점검

- `core/sensors/`에 센서 지원 여부를 표현하는 모델을 만든다.
- 가속도계, 자이로, 지자기, 기압계 스트림을 구독 가능한지 확인한다.
- 자이로가 없으면 정밀 실내 측위 미지원 상태로 처리한다.
- 기압계가 없으면 층 자동 감지를 비활성화하고 수동 층 선택으로 폴백한다.

### 2. 권한 플로우

- 위치 권한 요청 화면 또는 안내 상태를 만든다.
- 권한 거부 시 앱을 종료하지 않고 제한 모드와 재요청 경로를 제공한다.
- Android/iOS 권한 설명 문구가 실제 앱 기능과 맞는지 확인한다.

### 3. 상태 표시

- 앱 내부에서 `full`, `limited`, `unsupported` 같은 capability 상태를 구분한다.
- 지도 화면에 개발용 진단 패널 또는 간단한 상태 배지를 제공한다.
- 센서/권한 상태를 로그로 확인할 수 있게 한다.

## 수용 기준

- 앱 시작 시 센서와 위치 권한 상태를 확인한다.
- 자이로 미지원, 기압계 미지원, 위치 권한 거부 상황이 서로 다르게 처리된다.
- 지원되지 않는 기기에서도 앱이 크래시하지 않는다.
- README 또는 research 문서에 최소 지원 기기 조건이 연결되어 있다.
