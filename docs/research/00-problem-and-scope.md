# 00. 문제 정의와 범위

## 풀려는 문제

실외에서는 GPS로 길찾기가 잘 되지만, **건물 안으로 들어가는 순간 GPS가 끊겨 위치를 잃는다.**
대형 쇼핑몰·병원·공항·지하상가처럼 복잡한 실내에서 사람들은 "내가 지금 어디 있고, 목적지까지
어떻게 가야 하는가"를 알기 어렵다.

기존 실내 측위 솔루션은 대부분 **사전 인프라**를 요구한다.

- BLE 비콘 설치 (하드웨어 비용·유지보수)
- Wi-Fi 측위맵(fingerprint map) 사전 구축 (건물마다 일일이 측정)
- VPS용 3D 맵 서버 (대형 사업자만 가능)

이 프로젝트는 **그 인프라를 전부 제거**하고, 사용자 스마트폰에 이미 들어 있는 관성 센서(IMU)와
건물 평면도 한 장만으로 실내 위치 추적과 길찾기를 제공하는 것을 목표로 한다.

## 누구를 위한 것인가

- **1차 타겟**: 쇼핑몰·병원 등 복잡한 실내를 처음 방문한 일반인.
- 확장 가능성: 시각장애인 보행 보조, 대형 시설 직원 동선 관리 등(범위 밖, 발표 시 "향후 확장"으로 언급).

## 핵심 사용자 시나리오 (데모의 뼈대)

```
1. 사용자가 건물 밖에서 앱을 켠다 → GPS로 야외 위치 표시
2. 건물 입구로 걸어 들어간다
3. 앱이 "실내 진입"을 자동 감지 → 실내 평면도로 전환,
   입구 좌표를 현재 위치로 확정
4. 사용자가 걸어 다니면 PDR + 지도 매칭으로 실내 위치가 실시간 갱신
5. 목적지를 선택하면 평면도 위에 경로가 표시된다
```

이 "그냥 걸어 들어가면 자동으로 바뀐다"는 흐름이 시연의 임팩트 포인트다.

## 범위 (Scope)

### 포함 (In)

- 야외 GPS 추적 → 실내 자동 전환
- IMU 기반 PDR (걸음 감지 + 보폭 + 방향)
- 평면도 기반 Particle Filter 지도 매칭
- 단일 건물·평면도 기준 실시간 위치 렌더링
- 목적지까지의 실내 경로 표시 (평면도 그래프 위 최단 경로)
- 평면도/건물 데이터를 제공하는 백엔드 API

### 제외 (Out, 최소한 1차 버전에서는)

- 다건물·캠퍼스 단위 글로벌 측위
- Wi-Fi/BLE 측위 (인프라 의존이라 프로젝트 철학과 배치)
- VPS/카메라 기반 측위 (구현 난도·서버 비용 과다)
- 다층 자동 전환은 **기압계 있는 기기 한정**으로 선택 구현 (05 문서 참고)

## 성공 기준 (가설)

| 항목 | 목표치 | 근거 |
|---|---|---|
| 위치 오차 (복도 환경) | 95% CDF 기준 3~5m 이내 | 학술 사례에서 Map-Aided PF가 1~3m 달성 (03 문서) |
| 자동 전환 지연 | 입구 진입 후 수 초 이내 | GPS accuracy 급락 감지 기반 (04 문서) |
| 최소 동작 기기 | 자이로 탑재 스마트폰 | 가속도+자이로+지자기 필수 (05 문서) |
| 데모 완성도 | 입구→목적지 1회 무중단 시연 | 발표 임팩트 (08 문서) |

> 위 목표치는 사전조사 기반 **가설**이며, 프로토타입 측정 후 [08 평가 문서](08-evaluation.md)에서 갱신한다.

## 차별점 한 줄

> "비콘도, Wi-Fi 측위맵도, 서버 3D 맵도 없이 — 평면도 한 장과 폰 센서만으로 실내에서 길을 찾는다."
