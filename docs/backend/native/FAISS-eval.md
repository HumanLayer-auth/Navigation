### 임계값 0.50의 재현율 공백 (false negative)
라이브로만 보이는 실패다. 0.50 미만이라 no_match로 떨어지지만 사실 맞는 질의:

|---|---|---|---|
| `현금 뽑을 데` | 0.464 | ATM (하나은행) | ATM이 top인데 0.50 미달로 놓침 |
| `가방 보러 왔어` | 0.453 | 타임파리(잡화·액세서리) | 관련 매장이나 미달 |

### 왜 임계값을 더 낮추지 않는가 — 좁은 마진
- 무의미 질의: `asdfqwerzxcv` 0.458, `ㅋㅋㅋㅋㅋ` 0.437.
- 유효한 ATM(0.464)과 무의미(0.458)의 간격이 **0.006**뿐이다. 임계값을 0.46으로 낮추면 ATM은 잡지만 `asdfqwerzxcv`(0.458)도 통과해 **엉뚱한 매장을 답으로 내보낸다.**
- 길찾기에서는 **틀린 매장을 안내하는 것 > "다시 말해 주세요"** 가 더 나쁘다. 그래서 0.50은 **정밀도 우선** 선택이다(가방·ATM을 놓치더라도 헛매칭을 막는다).

### 재현율을 올리려면 (별도 단계)
절대 임계값 하나로는 한계가 명확하다. 다음이 근본 개선:
1. **임베딩 텍스트 보강** — 매장에 설명·동의어를 붙여 임베딩(예: ATM → "ATM 현금 인출 은행"). 이름만으로는 신호가 약하다.
2. **형태소(Kiwi) 정규화** — 질의 노이즈 제거로 점수 안정화.
3. **POI 시설 처리** — 에스컬레이터처럼 store에 없는 시설은 POI로 별도 검색(FAISS.md 9절의 이중표현 정리와 연결).
4. **마진/상대 기준** — 절대 임계값 대신 top1-top2 간격 등으로 무의미 질의를 더 잘 거른다.
