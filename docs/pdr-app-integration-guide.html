1. 기존 앱 구조

기존 Navigation Client는 화면 라우팅과 repository 중심으로 실외·실내 안내를 제공한다.

→
Arrival도착 처리
기존 실내 화면의 좌표 한계

IndoorMap과 RouteGuide는 현재 GeoJSON의 LatLng 모델과 더미 마커를 사용한다. PDR은 세션 시작점 기준 미터 좌표를 내므로, 변환 없이 기존 마커에 연결하면 안 된다.

2. PDR은 앱 어디에 붙었나

PDR은 특정 화면의 위젯이 아니라 앱 범위에서 살아 있는 headless 서비스다.

4. PDR이 제공하는 기능

Confirmed 위치·경로

PdrSnapshot.position, path, steps, distanceM을 제공한다. 세션 시작점 기준 로컬 미터 좌표다.

Preview 품질 신호

주황 accel preview는 독립적인 실시간 품질 신호다. 초록 위치와 평균내거나 자동 전환하지 않는다.

안전한 runtime 상태

시작·실행·pause·종료·오류 상태와 warning code를 UI에 제공한다. 오류 뒤 snapshot quality도 degraded가 된다.

Runtime state

idle안내·센서 세션이 꺼짐
starting첫 native 이벤트 대기
runningPDR 이벤트 처리 중
pausedbackground에서 센서 중지
stopping명시적 종료 처리 중
degraded권한·센서·채널 오류

5. anchor와 현재 제품 경계

PDR은 상대 좌표를 계산한다. 지도 위 실제 위치는 anchor와 floor 좌표계가 있어야 한다.

floorPoint = R(rotationDeg) × pdrPoint + anchorLocalM
PDR local point세션 시작점 기준
미터 좌표
+
Anchor시작 위치·층·회전
→
Floor local_m실제 층 지도 좌표
→
Meter-space rendererUI팀 렌더링
현재 가능한 것과 아직 하지 않는 것

센서 기반 상대 이동은 이미 동작한다. 그러나 IndoorMap과 RouteGuide는 아직 startGuidance()를 호출하지 않고 PDR marker도 그리지 않는다. 보정 전 위치를 지도에 그려 정확한 척하지 않기 위한 의도적인 경계다.
